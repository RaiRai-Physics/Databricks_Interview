1. What is Databricks?

Answer:
Databricks is a cloud-based data analytics platform built around Apache Spark. It is used for data engineering, data science, machine learning, streaming, SQL analytics, and lakehouse architecture. It lets teams build pipelines, notebooks, jobs, dashboards, and production workflows in one workspace.

Databricks’ lakehouse approach combines data lake flexibility with warehouse-style reliability and governance. Delta Lake is the default optimized storage layer for lakehouse tables on Databricks.

2. What is a lakehouse?

Answer:
A lakehouse is an architecture that combines features of a data lake and a data warehouse. Data is stored in low-cost object storage, such as S3, ADLS, or GCS, but supports structured tables, ACID transactions, schema enforcement, BI queries, and ML workloads.

In simple words:
Data lake = cheap storage
Data warehouse = reliable analytics
Lakehouse = both wearing one jacket.

3. What is Delta Lake?

Answer:
Delta Lake is an open-source storage layer used in Databricks. It adds reliability features to data lake files, especially Parquet files. It supports:

ACID transactions
Schema enforcement
Time travel
Scalable metadata handling
Upserts and deletes using MERGE, UPDATE, and DELETE

Delta Lake uses a transaction log to track table changes and provide consistency.

4. What are ACID transactions?

Answer:
ACID stands for:

Atomicity: transaction fully succeeds or fully fails
Consistency: data remains valid after the transaction
Isolation: concurrent users do not corrupt each other’s work
Durability: committed data is not lost

In Databricks, Delta Lake provides ACID guarantees, which makes data lake tables more reliable for production pipelines.

5. What is the difference between Parquet and Delta?

Answer:

Parquet	Delta
File format	Storage layer built on Parquet
No built-in transactions	Supports ACID transactions
No time travel by default	Supports time travel
No easy upsert/delete	Supports MERGE, UPDATE, DELETE
Good for storage	Better for production lakehouse tables

Interview line:
“Parquet is the file format. Delta adds transaction management, metadata, reliability, and table-level features on top of Parquet.”

6. What is a Delta table?

Answer:
A Delta table is a table stored in Delta Lake format. It contains data files, usually Parquet, plus a _delta_log transaction log. The log tracks table versions, schema, commits, and operations.

Example:

CREATE TABLE patients_delta
USING DELTA
AS SELECT * FROM patients_raw;
7. What is time travel in Delta Lake?

Answer:
Time travel lets us query an older version of a Delta table. This is useful for debugging, auditing, rollback, and comparing historical data.

Example:

SELECT * FROM claims VERSION AS OF 5;

or

SELECT * FROM claims TIMESTAMP AS OF '2026-04-01';
8. What is schema enforcement?

Answer:
Schema enforcement means Delta Lake checks whether incoming data matches the table schema before writing. If the data has unexpected columns or wrong data types, the write can fail. This prevents bad records from silently entering production tables.

9. What is schema evolution?

Answer:
Schema evolution allows the table schema to change when new columns arrive. For example, if a new column appears in source data, Databricks can evolve the table schema when configured.

Example:

df.write.format("delta") \
  .option("mergeSchema", "true") \
  .mode("append") \
  .saveAsTable("subscriber")
10. What is the medallion architecture?

Answer:
The medallion architecture organizes data into layers:

Layer	Meaning
Bronze	Raw data
Silver	Cleaned and validated data
Gold	Business-ready aggregated data

For example, in a healthcare insurance project:

Bronze: raw claims, patients, subscribers from S3
Silver: cleaned records with null handling and deduplication
Gold: use-case outputs like rejected claims, top disease, most profitable group

Databricks describes bronze, silver, and gold as progressively higher-quality layers in the lakehouse.
