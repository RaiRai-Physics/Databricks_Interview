31. What is MERGE INTO in Delta?

Answer:
MERGE INTO is used for upsert operations, meaning update existing rows and insert new rows.

Example:

MERGE INTO target_claims t
USING source_claims s
ON t.claim_id = s.claim_id
WHEN MATCHED THEN UPDATE SET *
WHEN NOT MATCHED THEN INSERT *

This is commonly used for incremental loading.

32. How do you read data from S3 in Databricks?

Answer:

df = spark.read.option("header", True).csv("s3://bucket/input-data/patients.csv")

For JSON:

claims = spark.read.json("s3://bucket/input-data/claims.json")

For Delta:

df = spark.read.format("delta").load("s3://bucket/clean-data/claims")

In production, access is usually managed with IAM roles, instance profiles, storage credentials, or Unity Catalog external locations.

33. How do you write data to Delta?

Answer:

df.write.format("delta") \
  .mode("overwrite") \
  .saveAsTable("project_core.claims")

Append mode:

df.write.format("delta") \
  .mode("append") \
  .saveAsTable("project_core.claims")
  
34. What is the difference between overwrite and append?

Answer:

Mode	Meaning
append	Adds new records
overwrite	Replaces existing data
ignore	Skips write if data exists
error	Fails if data exists

Example:

df.write.mode("append").saveAsTable("claims")

35. What is caching in Spark?

Answer:
Caching stores a DataFrame in memory or disk so it can be reused faster.

Example:

df.cache()
df.count()

Use caching only when the same DataFrame is reused multiple times. Do not cache everything, or the cluster becomes a storage closet with wheels.

36. What is lazy evaluation in Spark?

Answer:
Spark does not execute transformations immediately. It builds a logical plan and executes only when an action is called.

Transformations:

filter()
select()
groupBy()
join()

Actions:

count()
show()
collect()
write()

Example:

df2 = df.filter("amount > 1000")  # not executed yet
df2.count()                       # executed now

37. What is the difference between transformation and action?

Answer:

Transformation	Action
Creates a new DataFrame	Triggers execution
Lazy	Eager
Example: filter, select	Example: count, show, write

38. What is a notebook in Databricks?

Answer:
A notebook is an interactive development environment where we can write code in Python, SQL, Scala, or R. It is used for exploration, ETL development, debugging, visualization, and collaboration.

39. What are widgets in Databricks notebooks?

Answer:
Widgets allow parameter input in notebooks. They are useful for reusable jobs.

Example:

dbutils.widgets.text("input_path", "")
path = dbutils.widgets.get("input_path")

Use case: passing environment, file path, date, or table name into a job.

40. Scenario: A Databricks job is running slowly. What do you check?

Answer:
I would check:

Spark UI stages and tasks
Shuffle size
Skewed partitions
Input file sizes
Number of small files
Join strategy
Cluster size
Partition pruning
Delta table optimization
Whether unnecessary columns are being read

Then I would apply fixes such as filtering early, selecting fewer columns, using broadcast joins, optimizing Delta tables, repartitioning carefully, and reviewing cluster configuration.
